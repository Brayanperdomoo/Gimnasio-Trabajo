Arquitectura

Los 3 repos del trabajo

🗄️

Repo DB

PostgreSQL con Liquibase, Docker, datos iniciales, rollbacks y GitHub Actions para validar cambios.

⚙️

Repo Backend

API PHP con estructura por modelos, servicios, repositorios, controladores y documentación Swagger.

💻

Repo Frontend

Landing page y CRUD visual conectado al backend usando Axios.

Modelo de datos

5 entidades

  • Primarias: entrenadores y planes.
  • Secundarias: miembros, clases y reservas.
  • Relaciones: miembros depende de planes, clases depende de entrenadores, reservas depende de miembros y clases.
  • Reglas: llaves foráneas, checks de estado, cupos y precios positivos.

Cómo probar

Comandos principales

docker compose up --build

Frontend: http://localhost:3000
Swagger:  http://localhost:8080/docs
API:      http://localhost:8080/api

GitHub Actions

Validación antes del merge

1Push o Pull Request
2GitHub levanta PostgreSQL temporal
3Liquibase valida y ejecuta update
4Si falla, se bloquea el merge
